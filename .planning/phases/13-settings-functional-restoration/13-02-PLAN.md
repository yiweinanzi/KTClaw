  <name>Task 1: Wire startMinimized (autostart-only) and minimizeToTray (close→tray)</name>
  <files>electron/main/index.ts, electron/main/tray.ts, tests/unit/desktop-behavior.test.ts</files>
  <behavior>
    - When process.argv includes '--autostart' AND getSetting('startMinimized') is true, mainWindow.show() is NOT called at startup; window remains hidden
    - When process.argv does NOT include '--autostart', window always shows regardless of startMinimized value (per D-18)
    - When getSetting('minimizeToTray') is true, the 'close' event on mainWindow calls event.preventDefault() + mainWindow.hide() instead of allowing quit (per D-19)
    - When getSetting('minimizeToTray') is false, close behaves normally (allows quit flow)
    - The minimize button ('minimize' event) is never intercepted — it always performs a normal minimize
    - Existing single-instance behavior (second launch focuses existing window) must not regress (per D-03)
  </behavior>
  <action>
In electron/main/index.ts:

1. After creating mainWindow, read `const startMinimized = await getSetting('startMinimized')` and `const minimizeToTray = await getSetting('minimizeToTray')`.

2. Detect autostart: `const isAutostart = process.argv.includes('--autostart')`.

3. In the window ready-to-show / show logic: if `isAutostart && startMinimized`, skip the `mainWindow.show()` call (window stays hidden, tray icon is still created). Otherwise call `mainWindow.show()` as normal.

4. Wire close-to-tray: add a `mainWindow.on('close', (event) => { ... })` handler. If `minimizeToTray` is true AND `!isQuitting()`, call `event.preventDefault(); mainWindow.hide(); return`. Otherwise let the existing quit lifecycle proceed. This must be added AFTER the existing quit-lifecycle close handler so it does not interfere with the quit flow.

5. In electron/main/tray.ts, export a `sendDesktopNotification(title: string, body: string): void` helper that uses Electron's `Notification` API (import from 'electron'). Guard with `Notification.isSupported()`. This function will be called by Plan 04's notification emitters.

6. Write tests/unit/desktop-behavior.test.ts covering:
   - isAutostart=true + startMinimized=true → window.show not called
   - isAutostart=false + startMinimized=true → window.show IS called
   - minimizeToTray=true → close event calls preventDefault + hide
   - minimizeToTray=false → close event does not call preventDefault
  </action>

  </verify>
